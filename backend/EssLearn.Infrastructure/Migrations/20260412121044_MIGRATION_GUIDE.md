# Database Schema Guide

**All database tables are now defined in a single schema file: `backend/schema.sql`**

## Quick Start

To initialize or rebuild the entire database:

```bash
psql -U username -d database_name -f schema.sql
```

## Tables Included

1. **LearningFields** - Categories of learning content
2. **Channels** - YouTube channels
3. **Playlists** - Learning playlists with foreign key to LearningField
4. **Videos** - Individual videos in playlists
5. **VideoProgress** - User progress tracking for videos
6. **DownloadedVideos** - Local copies of downloaded videos
7. **Roadmaps** - Learning roadmaps
8. **RoadmapNodes** - Nodes within roadmaps, linking to playlists
9. **BlobStorageLogs** - Audit logs for blob storage operations
10. **StorageIntegrity** - Integrity tracking for stored blobs

## Table Relationships

```
LearningFields
├── Playlists (FK: FieldId → cascade delete)
│   ├── Videos (FK: PlaylistId → cascade delete)
│   │   ├── VideoProgress (FK: VideoId → cascade delete)
│   │   └── DownloadedVideos (FK: VideoId → cascade delete)
│   └── RoadmapNodes (FK: PlaylistId → cascade delete)
└── Channels (optional)
    └── Playlists (FK: ChannelId → set null)

Roadmaps
└── RoadmapNodes (FK: RoadmapId → cascade delete)
    ├── Self-referencing (FK: ParentId → set null)
    └── Playlists (FK: PlaylistId → cascade delete)
```

## Execution Methods

### Using psql CLI
```bash
psql -U postgres -d esslearn -f schema.sql
```

### Using pgAdmin
1. Right-click database → Query Tool
2. Open file → select schema.sql
3. Execute

### Using Docker
```bash
docker exec -it esslearn-db psql -U postgres -d esslearn -f /app/schema.sql
```

### Using DBeaver
1. New SQL Script
2. Open schema.sql
3. Execute All

## Notes

- All tables use PostgreSQL SERIAL (auto-increment) for IDs
- All tables include `CreatedAt` and `UpdatedAt` timestamps
- Foreign keys enforce referential integrity
- Indexes are created on commonly queried columns
- The script uses `IF NOT EXISTS` to safely run multiple times
